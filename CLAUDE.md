# Mindset Logos

This repo is the source of truth for every Mindset Consulting customer logo. Each customer has an `on-light` variant (dark logo, for cream backgrounds) and an `on-dark` variant (light logo, for navy backgrounds). Both are transparent, hand-verified, and live in `public/logos/`.

The gallery is a Next.js app deployed to Vercel under the `mindsetconsulting` team. The MCP server (`mcp/`) exposes the library to Claude Code sessions across Mindset.

## Working on the library

```bash
bun run dev           # start the gallery at localhost:3000
bun run audit         # re-scan public/logos/ and rewrite audit.json
bun run fetch         # pull new/missing logos from Brandfetch
bun run fix           # auto-fix flat / icon / tiny variants
bun run apply-approvals   # apply approvals.json decisions
```

Data files in `public/logos/`:
- `manifest.json` — the canonical list of 119 customers (slug, name, website, industry, verticals)
- `audit.json` — per-variant quality report (generated by `audit.ts`)
- `.pending-approval.json` — slugs Claude has fixed but the user hasn't blessed yet
- `.approved-overrides.json` — slugs the user has approved even though the heuristic flags them (square/icon marks)

## Adding a new customer

1. Add an entry to `public/logos/manifest.json` with slug, name, website, industry, verticals.
2. `bun run fetch <slug>` — fetches both variants from Brandfetch.
3. `bun run audit` — check quality.
4. If the fetch didn't produce clean variants, use the techniques below to re-source from the vendor's website.
5. Add slug to `.pending-approval.json`, re-run audit, review in the gallery, remove from pending when approved.

## The three logo-fix techniques

These techniques were earned the hard way on 119 real customer logos. If you're fixing a variant, pick the one that matches the source state.

### Technique 1: Trust existing alpha

**When:** Brandfetch WebPs (and many other raster sources) often already have transparent regions stored as `srgba(0,0,0,0)` — RGB=(0,0,0) with alpha=0. Invisible when composited, but **dangerous**: if you run `-alpha off` on them, the transparent-black becomes **opaque black** and creates a visible rectangle around the logo.

**Rule:** inspect the corner pixel first. If it reads `srgba(0,0,0,0)`, the file is already transparent — just `-trim +repage` and save. Do **not** touch alpha.

```bash
magick <file> -format "0,0=%[pixel:p{0,0}] center=%[pixel:p{W/2,H/2}]\n" info:
# If 0,0 is srgba(0,0,0,0), this file is already transparent.
magick <file> -trim +repage -bordercolor none -border 20 output.png
```

This burned us on 3M multiple times before we realized the file was already correct — we were fighting alpha we already had.

### Technique 2: Pillow pixel swap (raster with visible theme colors)

**When:** The source is a PNG/WebP with a transparent background and you need the opposite-theme version. For example, a dark-text wordmark on transparent, and you need a white-text version for dark backgrounds. Or you have a colored emblem + dark text, and need to swap only the dark text to white while keeping the emblem.

**Why not ImageMagick `-opaque` / `-transparent`:** Fuzz matching in RGB space leaves dark anti-aliased outlines around edges. Pillow lets you express the exact per-pixel rule and preserves anti-aliasing by keeping each pixel's original alpha while only rewriting RGB.

**Rule:** identify the color axis that separates "text" from "emblem" — is text `b > r + 25` (navy vs orange)? Is text `r<50 and g<50 and b<50` (black vs teal)? Is it `max(r,g,b) < 80` (any dark color vs brand color)? Write the condition, walk pixels, replace RGB with the target color, **keep the original alpha**.

```python
from PIL import Image
img = Image.open(src).convert('RGBA')
px = img.load()
w, h = img.size
for y in range(h):
    for x in range(w):
        r, g, b, a = px[x, y]
        if a == 0:
            continue
        if max(r, g, b) < 80:  # near-black → white, keep brand colors
            px[x, y] = (255, 255, 255, a)  # keep original alpha!
img.save(out)
```

Per-logo tuning: Agiliti uses `b > r + 25` to catch navy text while preserving orange arc. Johnson Screens uses `r<50 and g<50 and b<50` to catch black text while preserving teal emblem. Sullair uses `g > r and g > b` to catch green on white. Always sample the source and pick the simplest axis that distinguishes text from everything else.

**Critical gotcha on verification:** Do not trust `magick -background "#0A1628" input.png -resize x200 out.png` — at small sizes it can render legitimate anti-aliased text as a solid white block, making you think the pixel walk failed. Always verify with Pillow's own compositing:

```python
img = Image.open(out).convert('RGBA')
bg = Image.new('RGB', img.size, (10, 22, 40))
bg.paste(img, mask=img.split()[3])
bg.save('/tmp/check.png')
```

If Pillow's composite looks clean but ImageMagick's doesn't, trust Pillow.

### Technique 2b: Distance from white (raster with baked-in solid background)

**When:** The source is a raster (usually from `logovectorseek.com`, `seeklogo.com`, or a vendor PNG) with a solid white background and no alpha channel. You need to recover the alpha.

**Rule:** set each pixel's alpha based on its distance from pure white. Pure white → transparent. Mid-tones → partial alpha (smooth edges). Saturated colors → fully opaque.

```python
from PIL import Image
img = Image.open(src).convert('RGBA')
px = img.load()
w, h = img.size
for y in range(h):
    for x in range(w):
        r, g, b, a = px[x, y]
        dist = max(255-r, 255-g, 255-b)  # how far from pure white
        if dist < 8:                       # pure white → transparent
            px[x, y] = (r, g, b, 0)
        elif dist < 40:                    # edge ramp → smooth anti-alias
            alpha = int((dist - 8) * 255 / 32)
            px[x, y] = (r, g, b, alpha)
        # else keep fully opaque
img.save(out)
```

Tune the thresholds: bump `< 8` up to `< 20` for tinted/cream backgrounds. Bump `< 40` up for thicker anti-aliased edges.

Used successfully on Durag (900x500 from logovectorseek → blue "DURAG" + gray "GROUP" with transparent BG) and Sun Chemical (600x600 from seeklogo → full color wordmark with DIC badge).

### Technique 2c: Flood-fill from corner (logos with internal same-color elements)

**When:** The source has a baked-in solid background **and** the interior contains elements the same color as the background (e.g., white "TORO" text inside a red rounded rectangle — both outer BG and letters are white). Distance-from-white would kill the letters. But if the letters are geometrically isolated from the outer BG by the enclosing shape, flood-fill from the corner pixel only touches pixels reachable from that point — interior letters survive.

```bash
magick input.png \
  -alpha on \
  -bordercolor white -border 2 \
  -fill none -fuzz 5% \
  -floodfill +0+0 white \
  -shave 2x2 \
  -trim +repage \
  -bordercolor none -border 20 \
  output.png
```

**When NOT to use:** if the background connects to interior elements without any enclosing shape, or if there are transparent gaps in the shape that would let the fill reach protected interior regions.

Used successfully on Toro (600x600 seeklogo → sharp red rectangle + white TORO text preserved), BMW (400x400 → 440x440 with interior blue/white quadrants preserved), and Kodak (yellow corner artifacts removed while preserving the internal yellow K-chevron).

### Technique 3: SVG color swap (vendor SVG with white text)

**When:** The vendor's SVG is designed for dark backgrounds (text paths use `fill="white"` or `fill="#ffffff"`) and you need an on-light variant. **Don't** do a naive global replace — the emblem often contains white paths too (inner circles, accents, decorative shapes meant to punch through a colored background). A global swap turns the emblem black and ruins it.

**Rule:** use a Python regex that only swaps `fill="white"` on paths whose starting M x-coord is in the text region (typically x > 80 if emblem is on the left).

```python
import re
def swap(m):
    attr = m.group(0)
    d = re.search(r'd="M\s*(-?[0-9.]+)', attr)
    if d and float(d.group(1)) > 80:  # text region
        return attr.replace('fill="white"', 'fill="#1d1d1b"')
    return attr
result = re.sub(r'<path[^/]*?fill="white"/>', swap, svg_content)
```

Keep the original SVG as the on-dark variant. Save the swapped version as the on-light variant. Inspect the SVG fills first (`grep -oE 'fill="[^"]*"' file.svg | sort -u`) to pick the x boundary between emblem and text.

Got bitten on American Crystal Sugar: the first attempt used `sed 's/fill="white"/fill="#1d1d1b"/g'` and darkened both the wordmark AND a white-filled path inside the emblem, producing a black circle behind the sugar-beet icon. The x-coord filter fixes this.

## Sourcing new logos

Priority order when Brandfetch doesn't have a clean variant:

1. **Vendor website header** — `curl -sS -L -H "User-Agent: Mozilla/5.0 …" https://vendor.com/ | grep -oE 'src="[^"]*\.(svg|png|webp)[^"]*"'`. Pick the header/nav logo path. Often gives you an SVG directly.
2. **logovectorseek.com** — rasterized SVG dumps at 900x500+. Use the **distance-from-white** technique on these.
3. **seeklogo.com** — search: `https://seeklogo.com/search?q={brand}`. Click through to the vector-logo page and grab `images.seeklogo.com/logo-png/...-seeklogo-{id}.png`. Usually 600x600. Apply **distance-from-white** or **flood-fill-from-corner** depending on whether interior elements share the BG color.
4. **Wikipedia/Wikimedia Commons** — search `https://commons.wikimedia.org/w/api.php?action=query&format=json&list=search&srsearch={brand}+logo&srnamespace=6`.
5. **Vendor press/media page** — try `/about-us/media/`, `/press/`, `/brand/`, `/downloads/`.

Always fetch with a full browser User-Agent and a Referer header matching the site — many CDNs (Incapsula, Cloudflare) will 403 a bare `curl`.

## Audit and approval workflow

- `audit.ts` walks `public/logos/`, classifies each variant (`good`, `icon`, `flat`, `tiny`, `missing`), and writes `audit.json`.
- Slugs in `.pending-approval.json` are forced to `status: 'partial'` so they stay in the review queue even after a fix makes them technically `all-good`. This prevents "ready-to-ship" drift while you're iterating.
- Slugs in `.approved-overrides.json` are forced to `status: 'all-good'` regardless of heuristic. Use this for genuinely square marks (BMW roundel, Home Depot orange square) where the `icon` flag is a false positive.
- When you fix a logo, add it to `.pending-approval.json` and re-run audit. When the user approves it, remove it from pending.

## Never do these

- Don't run `-alpha off` on a file you haven't confirmed is opaque. Transparent-black pixels become visible black boxes.
- Don't trust `magick -background ... -resize` for verification at small sizes. Use Pillow compositing.
- Don't do a global `fill="white"` swap on vendor SVGs without an x-coord filter.
- Don't delete a logo's file without also pruning the corresponding `manifest.json` entry — the audit will flag it and the MCP will serve broken links.
